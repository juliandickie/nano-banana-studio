## Key Design Decisions

### Logo exclusion
Never mention "logo" in Presentation prompts — Gemini generates unwanted logo artifacts. Describe the area as "clean negative space" instead. Logos are composited in Keynote/PowerPoint/Slides.

### Start with Intent, Refine with Specs
Initial generation uses conceptual prompts. Follow-up refinements add PEEL specs (camera, materials, angles). This matches Nano Banana 2's conversational editing design.

### Fallback chain order
MCP (primary) → Direct Gemini API (if `GOOGLE_AI_API_KEY` set) → Replicate (if `REPLICATE_API_TOKEN` set). Google AI Studio first because it's free and direct.

### API key storage
Both keys stored in `~/.banana/config.json`. Scripts check: CLI flag → env var → config file. The Google key is also in `~/.claude/settings.json` for the MCP server. This solved the problem of fallback scripts not being able to find keys.

### Conversational setup
`/banana setup` uses Claude to guide users step-by-step instead of `input()` prompts (which don't work in Claude Code's shell). The script is called non-interactively with `--key` flag.

### Presentation mode: two options
- **Complete Slide** — model renders text (good for simple slides, leverages 94% text accuracy)
- **Background Only** — clean background with "NO text, NO logos" (for layering in presentation software)
